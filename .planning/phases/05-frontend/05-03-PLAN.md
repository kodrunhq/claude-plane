3. Create `internal/server/frontend/embed.go`:
   ```go
   package frontend

   import (
       "embed"
       "io/fs"
       "net/http"
       "strings"
   )

   //go:embed dist/*
   var FrontendFS embed.FS

   // NewSPAHandler returns an http.Handler that serves the embedded frontend.
   // For paths that match a static file (JS, CSS, images), it serves the file directly.
   // For all other paths, it serves index.html to support client-side routing.
   func NewSPAHandler() http.Handler {
       stripped, err := fs.Sub(FrontendFS, "dist")
       if err != nil {
           panic("frontend: failed to create sub filesystem: " + err.Error())
       }
       fileServer := http.FileServer(http.FS(stripped))

       return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
           // Try to serve the exact file
           path := strings.TrimPrefix(r.URL.Path, "/")
           if path == "" {
               path = "index.html"
           }

           // Check if the file exists in the embedded FS
           if _, err := fs.Stat(stripped, path); err != nil {
               // File doesn't exist — serve index.html for SPA routing
               r.URL.Path = "/"
           }

           fileServer.ServeHTTP(w, r)
       })
   }
   ```

4. The package is named `frontend` and exports:
   - `FrontendFS` — the raw embed.FS (for cases where the server needs direct access)
   - `NewSPAHandler()` — ready-to-mount http.Handler

5. The SPA handler logic:
   - Strips `/` prefix from URL path
   - Checks if the path matches an actual file in the embedded dist/
   - If yes: serves the file (JS bundles, CSS, assets, favicon)
   - If no: rewrites to `/` which serves index.html (React Router handles client-side routing)
   - This prevents 404s when directly navigating to `/sessions/abc123`

6. The server's Chi router should mount this handler LAST, after all `/api/v1/*` and `/ws/*` routes, so API calls are not caught by the SPA fallback. This mounting happens in the server's router setup (Phase 3), not in this package. Document this in a code comment.

7. Run `go vet ./internal/server/frontend/...` to verify the package compiles. Note: `go build` will fail if dist/ only has .gitkeep (embed expects at least the pattern to match), so the build verification must be done after `npm run build` populates the dist directory. For verification, first run `cd web && npm run build` then `go vet ./internal/server/frontend/...`.
