   - `UserClaimsKey` exported context key constant
   - `JWTAuthMiddleware(authSvc *auth.Service) func(http.Handler) http.Handler` — extracts Bearer token from Authorization header, calls authSvc.ValidateToken, injects claims into context. Returns 401 JSON error for missing header, invalid format, or failed validation.

     - Public routes group: POST /api/v1/auth/register, POST /api/v1/auth/login
     - Protected routes group (with JWTAuthMiddleware): POST /api/v1/auth/logout, GET /api/v1/machines, GET /api/v1/machines/{machineID}

  <behavior>
    - Register: valid request returns 201 with user_id and email; duplicate email returns 409; empty email or short password returns 400
    - Login: valid credentials return 200 with token, user_id, email, role; invalid credentials return 401; generic error message (no user enumeration)
    - Logout: revokes JWT, returns 200; subsequent request with same token returns 401
    - ListMachines: returns 200 with machine array; overlays live status from connection manager
    - All protected endpoints return 401 without Authorization header
  </behavior>
  <action>
1. Create `internal/server/api/auth_handler.go`:
   - `(h *Handlers) Register(w, r)`:
     - Decode JSON body: email, password, display_name
     - Validate: email non-empty, password >= 8 chars
     - Hash password with store.HashPassword (the function from Phase 1 users.go)
     - Create User struct with uuid.New() for UserID, role="user"
     - Insert via store method (add CreateUser to store if not present — INSERT INTO users)
     - On UNIQUE violation: 409 "email already registered"
     - On success: 201 with user_id and email
   - `(h *Handlers) Login(w, r)`:
     - Decode JSON body: email, password
     - GetUserByEmail — if not found, return 401 "invalid credentials" (same message for not-found and wrong-password to prevent user enumeration)
     - VerifyPassword — if false, return 401 "invalid credentials"
     - IssueToken — return 200 with token, user_id, email, role
   - `(h *Handlers) Logout(w, r)`:
     - GetClaims from context
     - RevokeToken with claims.ID and claims.ExpiresAt.Time
     - Return 200 {"message": "logged out"}
   - Add `(s *Store) CreateUser(user *User) error` to `internal/server/store/users.go` if not already present: INSERT INTO users (user_id, email, display_name, password_hash, role) VALUES (?, ?, ?, ?, ?)
   - Add `isUniqueViolation(err error) bool` helper — check for SQLite UNIQUE constraint error string

2. Create `internal/server/api/machine_handler.go`:
   - `(h *Handlers) ListMachines(w, r)`:
     - Query store.ListMachines() for all machines from DB
     - For each machine, check connMgr.GetAgent(machineID) — if connected, override status to "connected" and set LastSeenAt to now
     - Return 200 with JSON array
   - `(h *Handlers) GetMachine(w, r)`:
     - Extract machineID from chi.URLParam(r, "machineID")
     - Query store.GetMachine(machineID)
     - Overlay live status from connMgr
     - Return 200 or 404

   - TestLogoutSuccess: login, logout with token, verify 200

   - TestListMachinesAuthenticated: login, GET /api/v1/machines with token, verify 200

   - TestProtectedEndpointNoAuth: GET /api/v1/machines with no header -> 401, with invalid token -> 401
