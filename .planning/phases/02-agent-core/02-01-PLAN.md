    - Event loop: single sender goroutine reads from sendCh and calls stream.Send (prevents concurrent Send), recv loop calls stream.Recv and dispatches commands
    - On successful connect, backoff resets
    - Keepalive: Time=30s, Timeout=10s, PermitWithoutStream=true
    - Test: backoff produces increasing delays up to max
    - Test: backoff resets to min
    - Test: agent connects to mTLS server, registers successfully
    - Test: agent reconnects after server restart (integration test with ephemeral server)
  </behavior>

   - `Backoff` struct with `current`, `min`, `max` durations
   - `NewBackoff(min, max time.Duration) *Backoff`
   - `Next() time.Duration` — doubles current, caps at max, adds 20% jitter using `math/rand/v2.Int64N`
   - `Reset()` — sets current back to min

2. Create `internal/agent/backoff_test.go` (write first):
   - `TestBackoffIncreasing`: 5 calls to Next(), verify each >= min and <= max, verify general increasing trend (allowing for jitter)
   - `TestBackoffCapsAtMax`: call Next() 20 times, verify all <= max + 20% jitter
   - `TestBackoffReset`: call Next() several times, call Reset(), verify next Next() is near min

3. Create `internal/agent/client.go`:
   - `AgentClient` struct: `cfg *config.AgentConfig`, `tlsCreds credentials.TransportCredentials`, `sessions SessionProvider` (interface for session manager — decoupled for Plan 02), `logger *slog.Logger`, `backoff *Backoff`, `mu sync.Mutex`, `conn *grpc.ClientConn`, `stream pb.AgentService_CommandStreamClient`, `connected bool`, `sendCh chan *pb.AgentEvent`
   - `SessionProvider` interface: `GetStates() []*pb.SessionState`, `HandleCommand(cmd *pb.ServerCommand)`, `StartRelay(sendCh chan<- *pb.AgentEvent)`, `StopRelay()` — implemented in Plan 02
   - `NewAgentClient(cfg *config.AgentConfig, sessions SessionProvider, logger *slog.Logger) (*AgentClient, error)` — builds TLS creds from config using `tlsutil.AgentTLSConfig`, creates backoff from config values (default 1s min, 60s max)
   - `Run(ctx context.Context) error` — reconnect loop: call `connectAndServe(ctx)`, if ctx cancelled return, else log warning and wait with backoff
   - `connectAndServe(ctx context.Context) error`:
     1. `grpc.NewClient(cfg.Server.Address, grpc.WithTransportCredentials(tlsCreds), grpc.WithKeepaliveParams(...))`
     2. `client.Register(ctx, &pb.RegisterRequest{MachineId: cfg.Agent.MachineID, ...})`
     3. `client.CommandStream(ctx)` to open bidi stream
     4. Set connected=true, reset backoff
     5. Start sender goroutine (reads from sendCh, calls stream.Send)
     6. Start session relay via sessions.StartRelay(sendCh)
     7. Enter recv loop: `stream.Recv()`, dispatch via `sessions.HandleCommand(cmd)`
     8. On recv error: stop relay, close sendCh, wait for sender goroutine, set connected=false, return error
   - `waitWithBackoff(ctx context.Context) error` — select on timer vs ctx.Done
   - Use `sync.WaitGroup` for per-connection goroutines to prevent leaks (per research Pitfall 1)
