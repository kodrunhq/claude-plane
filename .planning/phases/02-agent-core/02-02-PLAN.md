  <name>Task 1: PTY session lifecycle — spawn, read, write, resize, cleanup</name>
  <files>internal/agent/session.go, internal/agent/session_test.go</files>
  <behavior>
    - NewSession(id, command, args, workDir, envVars, rows, cols): spawns process in PTY via pty.StartWithSize, starts readLoop goroutine, starts waitForExit goroutine
    - readLoop: reads from PTY in 4096-byte chunks, sends to outputCh (buffered channel cap=256), drops on full channel (no blocking)
    - WriteInput(data []byte): writes data to PTY fd
    - Resize(rows, cols uint16): calls pty.Setsize on PTY fd
    - waitForExit: calls cmd.Wait(), sets exit code and status ("completed" or "failed"), closes PTY fd, closes outputCh
    - Kill(signal string): sends signal to process (SIGTERM default, escalate to SIGKILL after 5s timeout)
    - Status() returns current status string: "running", "completed", "failed"
    - OutputCh() returns the read-only output channel
    - Test: spawn "echo hello" in PTY, read output from channel, verify contains "hello"
    - Test: spawn "cat" in PTY, write input "test\n", read output, verify echo
    - Test: spawn short-lived process, verify status transitions to "completed" and outputCh is closed
    - Test: resize does not error on running session
  </behavior>
  <action>
1. Add dependency: `go get github.com/creack/pty@latest`

2. Create `internal/agent/session.go`:
   - `Session` struct: `ID string`, `Status string` (protected by mu), `ptyFile *os.File`, `cmd *exec.Cmd`, `mu sync.Mutex`, `outputCh chan []byte` (buffered, cap 256), `startedAt time.Time`, `exitCode int`, `logger *slog.Logger`
   - `NewSession(id, command string, args []string, workDir string, envVars map[string]string, rows, cols uint16, logger *slog.Logger) (*Session, error)`:
     - Build `exec.Cmd` with command, args, workDir
     - Set `cmd.Env = os.Environ()` plus provided envVars
     - Call `pty.StartWithSize(cmd, &pty.Winsize{Rows: rows, Cols: cols})`
     - Initialize Session with status "running"
     - Launch `go s.readLoop()` and `go s.waitForExit()`
   - `readLoop()`: 4096-byte buffer, loop `ptyFile.Read`, copy data, non-blocking send to outputCh (select with default to drop if full), exit on read error
   - `WriteInput(data []byte) error`: write to ptyFile
   - `Resize(rows, cols uint16) error`: `pty.Setsize(s.ptyFile, &pty.Winsize{Rows: rows, Cols: cols})`
   - `Kill(sig string) error`: parse signal (default SIGTERM), send to process, start 5s timer goroutine that sends SIGKILL if process hasn't exited
   - `waitForExit()`: `cmd.Wait()`, lock mu, set exitCode and status, close ptyFile, close outputCh
   - `Status() string`, `ExitCode() int`, `OutputCh() <-chan []byte`, `ID() string` accessors (lock mu for status/exitCode)

3. Create `internal/agent/session_test.go` (write tests first):
   - `TestSessionSpawnAndRead`: spawn `/bin/echo hello`, read from OutputCh with timeout, verify output contains "hello", verify status eventually becomes "completed"

   - `TestSessionExitStatus`: spawn `/bin/true`, wait for outputCh to close, verify Status()=="completed" and ExitCode()==0. Then spawn `/bin/false`, verify Status()=="completed" (or "failed") and ExitCode()!=0
   - `TestSessionResize`: spawn `/bin/cat`, call Resize(40, 120), verify no error
   - Use `time.After` with generous timeout (5s) in all channel reads to prevent test hangs
   - Skip tests on CI if PTY not available: `if os.Getenv("CI") != "" { t.Skip("PTY tests require terminal") }`
