<objective>
Enhance the agent's session subsystem with scrollback persistence and attach/detach lifecycle. Every PTY session writes output to an asciicast v2 scrollback file. When a user attaches, the agent replays the scrollback in chunks before switching to live output. Detach stops the live relay without killing the PTY.

Purpose: This is the agent-side foundation for session persistence (SESS-06) and reconnection (SESS-02). Without scrollback, attaching to a running session would show a blank terminal. Without proper detach, closing the browser would kill the session.
Output: ScrollbackWriter, enhanced Session with scrollback integration, enhanced SessionManager with attach/detach handling.
</objective>

   - `ScrollbackWriter` struct: `file *os.File`, `mu sync.Mutex`, `offset int64`, `started time.Time`
   - `NewScrollbackWriter(path string, cols, rows uint32) (*ScrollbackWriter, error)`:
     - `os.Create(path)`
     - Write asciicast v2 header: `{"version":2,"width":COLS,"height":ROWS,"timestamp":UNIX_TS}` followed by newline
     - Track the byte offset (header length)
   - `WriteOutput(data []byte) error`:
     - Lock mu
     - Calculate elapsed seconds: `time.Since(w.started).Seconds()`
     - Use `json.Marshal(string(data))` to properly escape the data string (handles binary, control chars, quotes)
     - Format line: `[ELAPSED, "o", ESCAPED_DATA]` — note ESCAPED_DATA already includes surrounding quotes from json.Marshal
     - Write line + newline to file
     - Increment offset by bytes written
   - `CurrentOffset() int64`: lock mu, return offset
   - `Close() error`: close file
   - `ReadScrollbackChunks(path string, chunkSize int) ([]ScrollbackChunk, error)`:
     - `ScrollbackChunk` struct: `Data []byte`, `Offset int64`, `IsFinal bool`
     - Open file, read in chunkSize-byte chunks
     - Each chunk gets the byte offset where it starts
     - Last chunk gets `IsFinal: true`
     - Return slice of chunks (simple approach — files are capped at reasonable sizes)
