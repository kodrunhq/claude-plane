<objective>
Build the server-side session management layer: SQLite session store, in-memory session registry for routing agent events to WebSocket subscribers, REST API handlers for session lifecycle, and the WebSocket terminal bridge that connects browser terminals to agent PTYs.

Purpose: This is the server-side relay that connects browsers to agents. Without this, there is no way to create sessions via the UI, no way to route terminal data, and no way to bridge WebSocket to gRPC. This plan delivers the complete server-side data path for terminal streaming.
Output: Session store, session registry, REST handlers, WebSocket bridge, flow control. Wired into the existing chi router.
</objective>

3. Create `internal/server/session/ws.go`:
   - `HandleTerminalWS(store, connMgr, registry, authSvc, logger)` returns `http.HandlerFunc`
   - Inside handler:
     - Extract sessionID from chi.URLParam
     - Authenticate: extract token from query param `?token=` (WebSocket can't send Authorization header), validate via authSvc.ValidateToken
     - Look up session in store, get machineID
     - Get agent from connMgr
     - Upgrade to WebSocket: `websocket.Accept(w, r, &websocket.AcceptOptions{})` — binary mode
     - Subscribe to registry: `ch := registry.Subscribe(sessionID)`
     - defer: `registry.Unsubscribe(sessionID, ch)`, `conn.CloseNow()`
     - Send RequestScrollbackCmd to agent
     - **Writer goroutine** (single goroutine for thread-safe writes):
       - Read from subscriber channel
       - If IsControl: `conn.Write(ctx, websocket.MessageText, msg.Data)`
       - Else: `conn.Write(ctx, websocket.MessageBinary, msg.Data)`
       - On context cancel, exit
     - **Reader loop** (main goroutine):
       - `msgType, data, err := conn.Read(ctx)`
       - On error: send DetachSessionCmd to agent, return
       - If MessageBinary: send InputDataCmd to agent with data
       - If MessageText: parse JSON control message
         - `{"type":"resize","cols":N,"rows":N}` -> send ResizeTerminalCmd to agent
     - When scrollback replay completes, the agent sends is_final=true. The server forwards this, then the registry publishes a control message `{"type":"scrollback_end"}` so the frontend knows to enable input.
   - **Connection to gRPC event loop**: The existing AgentServiceServer.CommandStream (from Phase 3) receives AgentEvents from agents. It needs to be extended to check for SessionOutput and ScrollbackChunkEvent events and call `registry.Publish(sessionID, data)` or `registry.PublishControl(sessionID, controlMsg)`. Add a `SetRegistry(r *Registry)` method to AgentServiceServer so the session registry can be injected.
