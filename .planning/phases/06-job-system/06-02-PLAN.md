  <name>Task 1: Job and step REST handlers</name>
  <files>internal/server/handler/jobs.go, internal/server/handler/jobs_test.go</files>
  <behavior>
    - POST /api/v1/jobs with {name, description} returns 201 + Job JSON
    - GET /api/v1/jobs returns 200 + []Job JSON
    - GET /api/v1/jobs/:id returns 200 + JobDetail JSON (job + steps + deps)
    - GET /api/v1/jobs/:id with invalid ID returns 404
    - PUT /api/v1/jobs/:id with {name, description} returns 200 + updated Job
    - DELETE /api/v1/jobs/:id returns 204
    - POST /api/v1/jobs/:id/steps with {name, prompt, machine_id, working_dir} returns 201 + Step
    - PUT /api/v1/jobs/:jid/steps/:sid returns 200 + updated Step
    - DELETE /api/v1/jobs/:jid/steps/:sid returns 204
    - POST /api/v1/jobs/:jid/steps/:sid/deps with {depends_on} returns 201; validates DAG (no cycles)
    - POST /api/v1/jobs/:jid/steps/:sid/deps with cyclic dep returns 400 + error message
    - DELETE /api/v1/jobs/:jid/steps/:sid/deps/:dep_id returns 204
  </behavior>
  <action>
    Create `internal/server/handler/jobs.go` with a `JobHandler` struct that depends on `store.JobStore` and `orchestrator.ValidateDAG` function.

    Implement handlers following the project's existing Chi router patterns:
    - `ListJobs(w, r)` — GET /api/v1/jobs
    - `GetJob(w, r)` — GET /api/v1/jobs/{id}, extract id from chi.URLParam
    - `CreateJob(w, r)` — POST /api/v1/jobs, decode JSON body {name, description}, get createdBy from auth context (use a helper or middleware value)
    - `UpdateJob(w, r)` — PUT /api/v1/jobs/{id}
    - `DeleteJob(w, r)` — DELETE /api/v1/jobs/{id}
    - `AddStep(w, r)` — POST /api/v1/jobs/{id}/steps
    - `UpdateStep(w, r)` — PUT /api/v1/jobs/{jobID}/steps/{stepID}
    - `DeleteStep(w, r)` — DELETE /api/v1/jobs/{jobID}/steps/{stepID}
    - `AddDependency(w, r)` — POST /api/v1/jobs/{jobID}/steps/{stepID}/deps. After adding to DB, call ValidateDAG; if cycle detected, remove the edge and return 400
    - `RemoveDependency(w, r)` — DELETE /api/v1/jobs/{jobID}/steps/{stepID}/deps/{depID}

    JSON response helper: `respondJSON(w, status, data)` and `respondError(w, status, message)`.

    Add a `RegisterJobRoutes(r chi.Router, h *JobHandler)` function that mounts all routes under `/api/v1`.

    Test file uses httptest.NewRecorder + chi router. Create an in-memory SQLite with schema, create a real JobStore, and test the full HTTP roundtrip for each endpoint.
