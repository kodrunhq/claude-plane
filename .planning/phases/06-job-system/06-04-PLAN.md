
3. Inside the protected `r.Group` block (after session routes), add:
   ```go
   // Job system routes
   if jobHandler != nil {
       handler.RegisterJobRoutes(r, jobHandler)
   }
   if runHandler != nil {
       handler.RegisterRunRoutes(r, runHandler)
   }
   ```

Note: RegisterJobRoutes and RegisterRunRoutes use flat routes with full `/api/v1/...` paths, so they must be registered inside the `r.Route("/api/v1", ...)` closure BUT outside the path prefix (since they already include it). Looking at the existing code, the routes are registered with full paths like `r.Post("/api/v1/jobs", ...)`. This means they should be called on the TOP-LEVEL router `r`, not inside the `/api/v1` route group. However, they need JWT middleware.

The cleanest approach: call RegisterJobRoutes and RegisterRunRoutes on the top-level router but wrap them in a Group with JWT middleware. Add AFTER the `/api/v1` route block:

```go
// Job system routes (flat paths, JWT-protected)
if jobHandler != nil || runHandler != nil {
    r.Group(func(r chi.Router) {
        r.Use(JWTAuthMiddleware(h.authSvc))
        if jobHandler != nil {
            handler.RegisterJobRoutes(r, jobHandler)
        }
        if runHandler != nil {
            handler.RegisterRunRoutes(r, runHandler)
        }
    })
}
```

4. Update any existing call sites of `NewRouter` in test files to pass `nil, nil` for the two new params. Search for `api.NewRouter(` in test files and add the extra nil args.
