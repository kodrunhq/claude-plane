  <action>
    **Install dependencies first:**
    ```bash
    cd web && npm install @xyflow/react @dagrejs/dagre && npm install -D @types/dagre
    ```

    **1. API client** (`web/src/api/jobs.ts`):
    Implement `jobsApi` object with methods matching all REST endpoints (list, get, create, update, delete for jobs; addStep, updateStep, deleteStep; addDependency, removeDependency; triggerRun, listRuns, getRun, cancelRun, retryStep). Use the project's existing `request` helper from the API layer. Define TypeScript types: Job, Step, StepDependency, JobDetail, Run, RunStep, RunDetail, CreateJobParams, CreateStepParams, UpdateStepParams.

    **2. Zustand stores:**
    - `web/src/stores/jobs.ts`: Job definitions store with TanStack Query integration for server state. Actions: selectJob, selectStep.
    - `web/src/stores/runs.ts`: Run state store. Handles `run.step.status` WebSocket events to update step statuses in real time. Actions: setActiveRun, updateStepStatus.

    **3. DAG components:**
    - `StepNode.tsx`: Custom ReactFlow node rendering step name, status badge (color-coded: gray=pending, blue=running, green=completed, red=failed), and machine_id. Accept `selected` state for highlighting in editor mode. Dimensions: 180x60 fixed.
    - `StepEdge.tsx`: Custom ReactFlow edge with color based on status (gray default, green when source completed, animated dashed when source is running).
    - `DAGCanvas.tsx`: Wrapper around `<ReactFlow>` that:
      - Accepts `steps`, `dependencies`, `runSteps?` (optional, for run view) as props
      - Converts steps + deps to ReactFlow nodes + edges
      - Uses dagre for auto-layout (rankdir: 'LR', nodesep: 50, ranksep: 100)
      - In editor mode (`editable` prop): allows connecting nodes (creating dependency edges) and clicking nodes to select for editing
      - In run mode (`runSteps` provided): read-only, shows live status colors, no drag-to-connect
      - Registers StepNode and StepEdge as custom node/edge types via `nodeTypes` and `edgeTypes`

    **4. Job editor components:**
    - `JobMetaForm.tsx`: Form with name and description fields for job metadata. Controlled component with onChange callback.
    - `StepEditor.tsx`: Form for editing selected step: name, prompt (textarea), machine_id (select from available machines), working_dir, command (default "claude"), args. Save/delete buttons. Shows "Select a step" placeholder when no step selected.

    **5. Pages:**
    - `JobsPage.tsx`: Lists all jobs in a table/card layout. Each job shows name, step count, last run status, created date. "New Job" button navigates to JobEditor. Click job row to navigate to JobEditor with job loaded. "Run" button triggers a run and navigates to RunDetail.
    - `JobEditor.tsx`: Split layout -- left panel is DAGCanvas (editable), right panel is StepEditor. Toolbar at top with job name (inline edit), "Add Step" button, "Save" button, "Run" button. Adding a step creates it via API and adds node to DAG. Connecting two nodes via drag creates a dependency edge via API (with cycle validation -- show error toast if rejected). Selecting a node loads its config in StepEditor. Persist changes on Save.
    - `RunDetail.tsx`: Top section is RunDAGView (read-only DAG with live status). Below is a panel showing the selected step's terminal output (reuse existing terminal component with the step's session_id). Shows run status, started_at, elapsed time. "Cancel Run" button. Click on a failed step node shows "Retry" button. Subscribe to WebSocket `run.step.status` events to update RunDAGView in real time.

    **6. RunDAGView.tsx:**
    Thin wrapper: renders DAGCanvas in read-only mode with runSteps data. Passes step status to StepNode for color coding. On node click, calls onStepSelect callback so RunDetail can show that step's terminal.

    **7. Route registration:**
    Add routes to the app router (follow existing routing pattern):
    - `/jobs` -> JobsPage
    - `/jobs/new` -> JobEditor (create mode)
    - `/jobs/:id` -> JobEditor (edit mode)
    - `/runs/:id` -> RunDetail

    Import `@xyflow/react/dist/style.css` in the app's CSS entry point or in DAGCanvas.
